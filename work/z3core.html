Redesigning CLI based Sleep Staging Solution

Sample Preview of PSG data

Basic Info

Polysomnography is a comprehensive test used to diagnose sleep disorders. Traditional method of analyzing the sleep data collected in the test involves a person manually going through the entire data and scoring it.

z3core reduces the time spend on manual analysis by 10x

Current cloud based system allows sleep labs to upload their data on a cloud server and the data is analyzed by AI models. Once completed the scored report is sent to sleep labs

Polysomnography test setup

The Challenge

Design a solution for sleep scoring process which is more efficient, transperant and scalable

Understanding the problem

Stakeholder Interview

I started with Interviewing the stakeholder and understanding the issues in current system. This is how the current system functioned

A Technical Pause

Montage is a configuration file used by z3core to analyze the sleep data. Different studies require different montage configuration.

Everytime data is uploaded a montage setting is needed to analyze the data. In existing system the montage setting (JSON File) is created by a Neurobit Employee and the end user can use the same file for similar data.

From company's perspective

Issues with current system

  • Current solution is not scalable
  • Someone has to manually configure the montages
  • The CLI based tool used to process the data is not intuative
  • Need more secure system with proper user access control
  • The tool has steep learning curve

Interviewing End Users

Once I understood the business constrains and issues. I interviewed the power users of current system and got their feedback on how the system is and if there are any issues or feedback. Based on the interview I got this insights

Key Findings

Defining the Challenges

1. Streamline the process of uploading data to getting analyzed data

2. Eliminate the need for manual configuration from Neurobit Employee

3. Provide end user simpler means of configuring the montage

The Solution

Challenge #1

Streamline the process of uploading data to getting analyzed data

A web portal is designed where Sleep Labs can register and upload the edf data and get report easily.

Challenge #2

Eliminate the need for manual configuration from Neurobit Employee
