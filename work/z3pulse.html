Redesigning the
Sleep Recording App for z3pulse

The Challenge

Redesigning the sleep recording app and improving user experience

Overview

z3pulse is a Sleep Recording Wearable Device which monitors sleep and provides end user with truly actionable insights on sleep health.

My Role

User Interview

Competitor Analysis

Ideation

User Interface Design

Usability Test

Team Size

1 Full Time Designer and 2 Design Interns

2 App Developers

Understanding the business goals, constraints and scope of design

I started with stakeholder interview to get better idea of what problem the product is trying to solve and what is the reason and motivation behind redesigning the app. With this discussion we decided on followings things

Understanding the End User

In order to better understand their interaction with the app. I needed to understand their interaction with the overall product. I interviewed 4 people who had taken at least 7 sleep tests and monitored their interaction with the existing app and documented the findings.

Key Findings

Mapping User Journey

Why

To identify and better visualize pain points at every stage of interaction I opted to create a User Journey Map. This was my attempt of getting more holistic view of the entire process.

User Journey map for Sleep Recording Experience

How this helped?

I was able to identify areas where I could contribute not only to improving the existing app but also to contribute to overall Customer Experience

Heuristic Evaluation

Why

To deep dive into existing solution and pin point exact pain points in the app UI I decided to perform Heuristic Evaluation

We identified issues and prioratized them according their severity. P1 being High Prioty to P4 being Low Priority issue.

Exploring Competitor's solutions

To fully understand the problems these solutions are trying to address I started researching about competitors and compiled a list. I started going through the competitor apps who had similar functionality.

Website & App/Play store reviews

Website helped me in gathering the unique value proposition about competitor

App / Store Reviews of competitor who's app had similar functionality revealed few technical as well as usability issues we might encounter in future

Some Common Issues

Some of the apps had a lot of “Connectivity Issues” and user ended up not using the device.

“Not so intuative directions” on how to use the device

Insights and the “Report is very technical” and not easy to understand

Our Solution

Made sure technical issues are tested and provided clear steps to resolve the issues or get help

Included clear instructions and infographics in app and packaging design on how to wear and use the device

Redesigned existing technical report and made it much more user friendly by removing technical jargon

The Business Constraint

The newly introduced business constraint needed the app to be generic and to be compatible with B2B model.

Up until this point our plan was to redesign the existing UI. But we realized that won't be sufficient

Updating the Information Architecture

New Sub Challenge

"Update the Information Architecture in such a way that the solution will be easily pluggable into B2B client's ecosystem with minimal change in the user flow"

The Solution

"A QR code based system (Standard Licensing Model) where the end users can simply scan a QR code to take a sleep test"

Changes in existing Information Architecture

1. Login / Signup Removed

By Removing Login Signup we made sure that the app is generic and could be used by every client

2. Flexible Report Delivery

The Client has total control over the sleep report and can choose how to share it with the end user

New Information Architecture and Low Fidelity Design

Solution

Onboarding Screens

Clear Infographics

A lot of user had problems setting up the device. Including the clear infrographics on onboarding screens and pacakging helped them set it up without any hassel

Home Screen

Removed Login / Signup

Now user don't have to signup or login to take a sleep test. They can just use the QR code provided to start a test and get report on mail

Sleep Recording Screen

Updated Animation

I added breathing animation to the recording screen. User can sync their breathing with the animation and it helps them to relax and helps in falling asleep quicker

Clear Device Status

Previous version contained unclear feedback on whether the device was connected and functioning properly

Clean UI

Removed a lot of unncessary information from the recording screen making it clean and less confusing

Usability Test

Tested the new UI on 2 existing customer and 2 team members. Got feedback on few things, that needed addressing and iterated over the new design to come up with final UI
